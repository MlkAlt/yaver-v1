# E4 — Dilekçe bankası (data + HTML)
**Rol:** evrak-engineer · **Gate:** PASS (`npx tsc --noEmit` = 0)

## Ne yapıldı
Öğretmen dilekçe bankası için SADECE 2 yeni data/HTML dosyası. Curated küçük set (performans modülü mantığı: az ve net) — **4 dilekçe türü**. Öğretmen türü seçer, hazır gövde kalıbını ([...] ipuçlu) düzenler; çıktı resmi MEB dilekçe formatında (dikey A4, Times New Roman) PDF'e gider. Wiring/commit YOK.

Referans `evraklar/dilekce/dilekcesi.docx` placeholder/mock çıktı (içinde İspanyolca "Fecha no especificada" + jenerik talimat cümleleri) — kullanıcının uyardığı gibi "gerçekten dolu değil". Yalnızca **yapı/mantık** alındı (mazeret izni bölümleri: öğretmen bilgileri, izin süresi, sebep, devredilecek görevler, ekler), mock cümleler kopyalanmadı; gövdeler resmi dile göre sıfırdan yazıldı. Diğer dosya `10185348_YYretmen_DilekYe.doc` antiword olmadığı için okunamadı, atlandı.

## Dokunulan dosyalar (ikisi de YENİ)
- `src/data/dilekceSablon.ts` — tipler + veri + helper
- `src/data/dilekceHtmlSablon.ts` — HTML üretici

## Public API (şefin wiring'de kullanacağı)
**Tipler:**
- `DilekceTuru = 'mazeret' | 'ucretsiz' | 'nakil' | 'genel'`
- `DilekceAlan` — form alanı anahtarları: `'makam' | 'adSoyad' | 'gorev' | 'tc' | 'govde' | 'ekler' | 'imzaYeri' | 'tarih'`
- `DilekceSablonu` = `{ id, ad, aciklama, makamVarsayilan, govdeKalibi, alanlar }`
- `DilekceFormData` = `{ turu, adSoyad, gorev, tc?, tarih, makam, govde, ekler: string[], imzaYeri? }`

**Veri/fonksiyon:**
- `DILEKCE_SABLONLARI: DilekceSablonu[]` (4 tür)
- `getDilekceSablonu(turu): DilekceSablonu`
- `bosDilekceFormu(turu): DilekceFormData` — makam/gövde kalıptan önden dolu, kişisel alanlar boş
- `dilekceHtmlOlustur(form: DilekceFormData): string`

**4 tür:** `mazeret` (Mazeret İzni), `ucretsiz` (Ücretsiz/Aylıksız İzin), `nakil` (Nakil/Tayin Talebi), `genel` (Genel/Boş — öğretmen serbest yazar).

## Sıradaki role ne gerek — WIRING NOTU (şef: SablonDoldurma + Evraklarim)
**sablonId:** öneri tek `sablonId='dilekce'`. Diğer sablonlardan farkı: dilekçenin **alt-türü** var → sihirbazın 1. adımı `DILEKCE_SABLONLARI` içinden **tür seçimi** olmalı (4 kart/pill). Seçilince `bosDilekceFormu(turu)` ile state doldur. (Alternatif: her tür ayrı EvraklarimScreen kartı — ama tek kart + tür adımı önerilir.)

**Init:** `bosDilekceFormu(turu)` → `govde` = `govdeKalibi` (düzenlenebilir), `makam` = `makamVarsayilan`. Kişisel alanları OnboardingContext'ten besle (memory kuralı: form alanı eklemeden önce OnboardingContext + Supabase şeması kontrol) — `adSoyad`, `gorev` (branş → "X Öğretmeni"), `makam` (okul adı → "… MÜDÜRLÜĞÜNE").

**Form alanları (her sablon `alanlar` sırasını verir):**
| alan | girdi tipi | not |
|---|---|---|
| `makam` | tek satır | prefill; render'da turkceBuyuk ile BÜYÜK harfe çevrilir (kullanıcı normal yazabilir) |
| `adSoyad` | tek satır | OnboardingContext |
| `gorev` | tek satır | branş → "… Öğretmeni" |
| `tc` | tek satır, **opsiyonel** | yalnız mazeret/ucretsiz/nakil'de; genel'de `alanlar`da YOK |
| `govde` | **çok satırlı (multiline)** | en önemli alan; `govdeKalibi` prefill, öğretmen [..] kısımları düzenler |
| `ekler` | **çok satırlı** | her satır bir ek → `.split('\n')` → `string[]`; boşsa çıktıda EKLER bloğu gösterilmez |
| `imzaYeri` | tek satır, **opsiyonel** | yer/şehir; doluysa "Konya, 04.07.2026", boşsa sadece tarih |
| `tarih` | tek satır | default bugün "GG.AA.YYYY" |

**Çıktı:** `dilekceHtmlOlustur(form)` → mevcut desen `Print.printToFileAsync({html})` → `Print.printAsync({uri})`.

## Kalite kapısı
- `npx tsc --noEmit` = **0** (proje geneli).
- Örnek çıktı 2 senaryoyla üretilip yapı gözle doğrulandı: (1) mazeret — dolu (ekler+tc+yer), (2) genel — minimal (opsiyoneller boş → EKLER/TC/yer doğru şekilde gizlendi). Format: @page A4 dikey / margin 20mm, Times New Roman 12pt/1.5, makam ortalı-bold, gövde justify+indent, sağ alt imza bloğu (tarih→boşluk→ad→görev→[tc]→İmza), sol alt EKLER (1-,2-). Türkçe büyük harf doğru (İ/I).

## Açık sorular / riskler
- `imzaYeri` "imza yeri"ni **yer adı (şehir)** olarak yorumladım (tarih öneki). Şef aksini isterse kolayca değişir.
- Makam default'ları `[OKUL ADI] / [İL]` placeholder içerir — şef OnboardingContext'ten gerçek değerle doldurmalı; doldurmazsa çıktıda köşeli parantez görünür (bilinçli, "doldur beni" sinyali).
